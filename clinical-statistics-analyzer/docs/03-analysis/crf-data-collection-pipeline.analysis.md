## 5. Detailed Gap Findings

### 5.1 Missing Features (Design/Plan present, Implementation absent)

| # | Item | Design/Plan Location | Description | Impact |
|---|------|---------------------|-------------|--------|
| 1 | Extraction Audit Trail (F10) | Plan F10 | No dedicated audit trail tracking per-field extraction provenance (source file, page, extraction method, confidence). Only basic `crf_extraction.log` file logging exists. | Medium |
| 2 | `--use-llm` flag passthrough | Design CLI spec | The `--use-llm` CLI flag is parsed by argparse but never passed to `run_extraction()` or down to the `FieldExtractor`. The `use_llm` parameter exists in `run_extraction()` signature but is unused in the function body. | Medium |
| 3 | Consistency rules CR004-CR007 | `validation_rules.json` | Rules CR004 (death date > diagnosis date), CR005 (CR date > induction date), CR006 (HCT date > CR date), CR007 (relapse date > CR date) are defined in JSON but NOT implemented in `DataValidator._check_consistency_rule()`. Only CR001, CR002, CR003 are implemented. | High |

### 5.2 Added Features (Implementation present, Design absent)

| # | Item | Implementation Location | Description | Impact |
|---|------|------------------------|-------------|--------|
| 1 | `extractor_v2.py` / `ImprovedExtractor` | `CRF_Extractor/src/extractor_v2.py` | A second extractor with SPSS code mapping and OCR cleanup. Not referenced in design. Also not imported in `main.py` (main.py imports from `extractor.py`, not `extractor_v2.py`). | Low (unused) |
| 2 | Excel export | `CRF_Extractor/src/exporter.py:65` | `export_excel()` method and `.xlsx` output in `export_all()`. Design only specifies SPSS and CSV. | Low |
| 3 | `pdf_processor.py` and `docx_processor.py` as separate modules | `CRF_Extractor/src/` | Design describes a single `document_processor` component. Implementation splits into 3 files (orchestrator + 2 format-specific processors). | Low (good practice) |
| 4 | `export_existing_spss_variables()` utility | `CRF_Extractor/src/exporter.py:176` | Utility to extract variable metadata from existing SPSS files. Not in design. | Low |
| 5 | `spss_value_mapping` in field_mapping.json | `CRF_Extractor/config/field_mapping.json:6` | Bidirectional SPSS code mappings. Design mentions "SPSS variable relationships" but not this specific structure. | Low |

### 5.3 Changed Features (Design differs from Implementation)

| # | Item | Design | Implementation | Impact |
|---|------|--------|----------------|--------|
| 1 | Step 3 function call | `validate_and_report` function | `DataValidator` instantiated directly; `validate_and_report()` exists as standalone function in `validator.py` but is NOT called from `main.py` | Low |
| 2 | `python-docx` dependency | Not in design, implied by DOCX support | Not listed in `requirements.txt` but imported in `docx_processor.py` | Medium |
| 3 | `field_mapping.json` structure | Design says "Variable Name, Regex/Search String, Data Type" | Implementation has sections with fields containing `crf_field`, `variable`, `type`, `extraction_method`, `patterns`, `sps_code`, `required` | Low (richer than designed) |

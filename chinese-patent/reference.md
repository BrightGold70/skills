# 中国专利说明书撰写参考

各章节撰写要点、用语规范与常见错误。供 agent 在撰写或改写时查阅。

## 技术领域

- 用简明语言说明发明所属的技术领域，不写广告语。
- 常见表述：「本发明涉及……」「本申请涉及……」。
- 避免：过于宽泛（如「涉及机械领域」）或混入发明内容。

## 背景技术

- 介绍与本发明最接近的现有技术，指出现有方案的不足或待解决问题。
- 可引用现有专利或公知技术，客观描述，不贬低他人。
- 常见问题：与「发明内容」重复、缺少对现有缺陷的明确指认。

## 发明内容

- **要解决的技术问题**：针对背景技术中的不足，明确本发明要解决的问题。
- **技术方案**：概括解决该问题所采用的技术手段，可与权利要求 1 对应。
- **有益效果**：相对现有技术带来的效果（如性能、成本、可靠性等）。
- 用语宜与权利要求书协调，避免说明书与权利要求表述不一致。

## 附图说明

- 对各附图逐一简要说明，如「图1是……结构示意图」「图2是……流程图」。
- 附图编号与正文引用、图片文件名一致（图1、图2……）。
- 仅说明「是什么图」，不在此展开技术细节；细节放在「具体实施方式」。

## 附图的代码生成

当附图可由程序绘制时，用 Python 脚本生成 PNG 等文件，保存到 `images/`，再在 HTML 中引用。

- **matplotlib**：框图、曲线图、结构示意。图中文字（标注、图例、轴标签）使用英文。
- **graphviz**（`pip install graphviz`，另需安装 Graphviz）：流程图、框图、关系图；节点/边文字宜用英文。
- **diagrams**（`pip install diagrams`）：系统架构、模块关系图；输出 PNG 到 `images/`。
- **输出**：统一保存为 `images/图N.png`（N 与附图说明编号一致），脚本可在说明书 HTML 同级目录运行，或指定 `--output-dir images/`。
- **示例**：见 skill 内 `scripts/generate_figure_example.py`，用 matplotlib 生成框图并保存；可据此改写为实际专利附图。依赖见 skill 根目录 `requirements.txt`（matplotlib）。

## 具体实施方式

- 结合附图详细描述技术方案的实施方式，使本领域技术人员能够实现。
- 对附图标记（零件编号）在正文中应有对应说明。
- 可给出多个实施例或变形例，以支持权利要求范围。
- 常见问题：只有概括无具体结构/步骤；附图与文字不对应。

## 权利要求书（在 HTML 中的位置）

- 可单独放在同一 HTML 的 `<section id="claims">` 或文件末尾。
- **独立权利要求**：限定完整技术方案，包含必要技术特征。
- **从属权利要求**：引用在先权利要求，并进一步限定附加技术特征。
- 撰写顺序：独立权利要求（通常 1 项）在前，从属权利要求在后；从属项可引用多项在先权利要求（如「根据权利要求1或2所述的……」）。
- 用语应与说明书一致，避免出现说明书中未出现的术语。

## 摘要

- 概括发明要点：技术问题、主要技术方案、主要有益效果；一般 300 字以内。
- 可配一幅代表性附图（摘要附图）。
- 在 HTML 中可放在单独 `<section id="abstract">` 或文末。

## 用语与格式注意

- 统一使用「本发明」或「本申请」；同一篇内不要混用「本发明」与「本实用新型」除非确属实用新型专利。
- 数字、单位、符号与国家标准或行业习惯一致。
- 避免商业性、宣传性用语（如「领先」「最佳」）。

## 常见错误速查

| 问题 | 建议 |
|------|------|
| 说明书与权利要求用语不一致 | 先定权利要求用语，再在说明书中统一使用 |
| 附图说明与附图不对应 | 按图号顺序写附图说明，并与 images/ 中文件名一致 |
| 具体实施方式过于笼统 | 结合附图写清结构、连接关系或步骤 |
| 图片用 HTTP URL | 必须用本地路径，否则 pandoc 无法嵌入 docx |
| PDF 转换 | 使用 `scripts/html_to_pdf.py`，需安装 `playwright` |

## 格式与排版规范

为确保生成的 docx 文件符合中国专利电子申请的要求，必须遵守以下格式限制：

### 1. 正文文字
- **字体**：统一使用 **宋体 (SimSun)**、**仿宋 (FangSong)** 或 **楷体**。
- **字号**：**小四 (12pt)** 或 **四号 (14pt)**。字高应在 3.5mm 至 4.5mm 之间。
- **行间距**：建议使用 **1.5 倍行距** 或固定值 **25-30pt**。
- **颜色**：纯黑色，字迹清晰。
- **页边距**：左边和上边各留 25mm，右边和下边各留 15mm。

### 2. 表格格式
- **表名/标题**：位于表格上方，使用 **四号黑体 (SimHei)**。
- **表内文字**：使用 **五号 (10.5pt)** 或 **小五 (9pt)** 宋体。
- **边框**：使用实线边框。
- **限制**：复杂的合并单元格表格建议以图片方式提交，简单的 N×M 表格可直接使用 HTML `<table>`。

### 3. 图像（附图）规范
- **分辨率**：72 DPI 至 300 DPI 之间。
- **物理尺寸**：最大不得超过 **165mm × 245mm**。
- **图号标注**：图号（如“图1”、“图2”）应在 HTML 正文中以文字形式编写，**严禁**包含在图片像素内。
- **绘图字体**：图片内的标注文字应使用英文，字体清晰可辨。
- **格式**：推荐使用 PNG 或 JPEG。

### 4. 段落与章节
- 章节标题（如“技术领域”）应加粗或使用较大的字号（如三号或四号）。
- 段落首行缩进 2 个字符。
